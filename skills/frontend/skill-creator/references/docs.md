# OpenCode Skill Guidelines

- Use proper frontmatter YAML
- Keep scope accurate: [root, ui], [root, api], etc.
- Critical patterns go at the top
- Minimal, focused examples
- Commands section required
- Use local references; avoid web URLs

<!-- vds code recommendations check  -->

- Use `code` for commands and code snippets
- Use `link` for references to other skills
- Use `image` for screenshots
- Use `video` for screencasts
- Use `table` for tabular data
- Use `note` for important notes
- Use `warning` for warnings
- Use `tip` for tips
- Use `important` for important information
- Use `caution` for cautions
- Use `danger` for dangerous information
- Use `quote` for quotes
- Use `highlight` for highlighted text
- Use `spoiler` for spoilers
- Use `details` for details
- Use `summary` for summaries
- Use `tabs` for tabbed content
- Use `tab` for tab content
- Use `tab-item` for tab item content
